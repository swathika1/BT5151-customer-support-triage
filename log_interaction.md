# Log Interaction Skill

## Role
This is an organisational skill. The Python code inserts a complete record of each ticket classification into a SQLite database deterministically. This enables post-hoc analysis, debugging, and feedback loops.

## When to use
At the end of the serving pipeline after all routing decisions are made. After draft_response_node. Triggered once per customer message. Also triggered when feedback is logged (see apply_feedback_node).

## How to execute
1. Ensure SQLite database exists at `data/interactions.db` (created once during setup)
2. Ensure `interactions` table exists with schema (see Output format section)
3. Gather all interaction data from state:
   - conversation_id: unique identifier for this interaction
   - timestamp: ISO 8601 timestamp
   - original_message: customer's message (before translation)
   - detected_language: language detected by detect_language_node
   - translated_message: after translation (if applicable)
   - predicted_category: model's prediction
   - confidence_score: model's max probability
   - class_probabilities: JSON of all 10 class scores
   - routing_decision: auto_approved, pending_review, or escalate_to_supervisor
   - routing_rationale: explanation of routing decision
   - requires_human_review: boolean
   - response_template: suggested response
   - agent_feedback: null initially (populated later by apply_feedback_node)
   - is_correct: null initially (marked by human agent later)
4. Insert record into `interactions` table
5. Retrieve inserted row id (auto-increment primary key)
6. Store interaction_id in state for later feedback collection
7. Log to state.messages: "[log_interaction] Logged interaction_id={interaction_id}"

## Inputs from agent state
- conversation_id: str - unique ID for this conversation (UUID or timestamp-based)
- customer_message: str - original message from customer
- translated_message: str - English message after translation (same as original if English)
- detected_language: str - language code detected
- predicted_label: str - the model's prediction
- confidence_score: float - confidence (0.0-1.0)
- class_probabilities: dict - all 10 class probabilities
- routing_decision: str - auto_approved, pending_review, escalate_to_supervisor
- routing_rationale: str - explanation
- requires_human_review: bool
- response_template: str - suggested response
- timestamp: datetime - when the interaction occurred
- selected_model_name: str - which model was used (e.g., "Linear SVM")

## Outputs to agent state
- interaction_id: int - primary key of logged interaction (for future reference/feedback)
- log_status: str - "success" or "failed"
- log_message: str - error message if failed
- database_path: str - path to SQLite database file

## Database Schema
```sql
CREATE TABLE IF NOT EXISTS interactions (
    interaction_id INTEGER PRIMARY KEY AUTOINCREMENT,
    conversation_id TEXT NOT NULL,
    timestamp DATETIME NOT NULL,
    original_message TEXT NOT NULL,
    detected_language TEXT,
    translated_message TEXT,
    predicted_category TEXT NOT NULL,
    confidence_score REAL,
    class_probabilities_json TEXT,  -- JSON string of all probs
    routing_decision TEXT,
    routing_rationale TEXT,
    requires_human_review BOOLEAN,
    response_template TEXT,
    model_name TEXT,
    agent_feedback TEXT,  -- filled in later by apply_feedback_node
    is_correct BOOLEAN,   -- filled in later by agent validation
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_timestamp ON interactions(timestamp);
CREATE INDEX idx_category ON interactions(predicted_category);
CREATE INDEX idx_routing ON interactions(routing_decision);
```

## Output format
Appends to state.messages:
"[log_interaction] Logged interaction_id=12847 for customer message 'I can't log in'. Routing: auto_approved. Stored in data/interactions.db."

## SQLite Connection
```python
import sqlite3
conn = sqlite3.connect('data/interactions.db')
cursor = conn.cursor()
# Run INSERT query
conn.commit()
conn.close()
```

## Notes
- interaction_id is auto-generated by SQLite; use it to link feedback later
- conversation_id can be generated once per customer session (to group multiple messages from same customer)
- Storing probability distribution (class_probabilities_json) enables later analysis of model confidence patterns
- is_correct will be set by human agents or feedback_node after they validate the prediction
- Database grows indefinitely; implement retention policy (e.g., delete interactions > 2 years old)
- For large-scale systems: consider offloading old interactions to data warehouse or BigQuery
- Log every interaction, good or bad (don't filter); the failures are most valuable for improvement
- Use this data to retrain the model periodically (see retrain_from_feedback_node)
